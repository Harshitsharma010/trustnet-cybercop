# Product

## Register

product

## Users

Recruiters, interviewers, and technical reviewers use TrustNet CyberCop to judge a portfolio project quickly. They need to understand that the system is a real cloud-connected phishing URL scanner, not a decorative AI mockup. The primary task is to paste or select a URL, run a scan, and inspect the verdict, reasons, model evidence, and deployment path.

## Product Purpose

TrustNet CyberCop demonstrates a practical security workflow: URL validation, feature extraction, lightweight ML inference, explainable risk scoring, scan history, and an AWS hosted dashboard/API path. Success means a reviewer can trust the interface, understand the engineering work, and see credible evidence without reading the entire repository first.

## Brand Personality

Credible, operational, precise. The interface should feel like a small security operations console built by an engineer who understands cloud and threat detection.

## Anti-references

Avoid generic AI dashboard styling: neon blue grids, glossy glass panels, oversized SaaS hero type, vague "AI-powered" positioning, decorative glow lines, and repeated card templates. Avoid implying this is a commercial production security product; it is a serious deployed portfolio project.

## Design Principles

Show evidence before decoration.

Make scanning feel like an operational workflow, not a marketing hero.

Use motion to reveal state, progress, and feedback.

Keep risk, model, API, and AWS proof visible at a glance.

Balance portfolio polish with honest product scope.

## Accessibility & Inclusion

Target WCAG AA contrast, keyboard-visible focus, readable dense data, responsive tables, and reduced-motion alternatives for all interface motion.
